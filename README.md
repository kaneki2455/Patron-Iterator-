# Patrón Iterator - Control de Asistencia RRHH

## Descripción

Este proyecto nació como parte del curso de Diseño de Software. La idea era
aplicar el Patrón Iterator a un caso real, y elegí el control de tardanzas
del área de Recursos Humanos porque me pareció el más cercano a algo que
se usaría en una empresa de verdad.

El sistema recorre los registros de asistencia de los empleados usando su
DNI como identificador, mostrando información como el nombre, fecha, hora
de ingreso y minutos de retraso — todo esto sin tocar directamente el
arreglo interno donde se guardan los datos. Eso es exactamente lo que
propone el patrón.
